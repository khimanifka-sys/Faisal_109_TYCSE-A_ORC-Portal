# 3 — Database design (ER plan)

Entities (tables):

1. `students` — main student + result data
2. `categories` — list of categories & their cutoffs (optional)
3. `result_logs` — (optional) record who viewed result when
4. `admins` — admin users for uploading results (optional)

Relationships:

* `categories.category_code` 1 — ∞ `students.category`
* `students.roll_no` 1 — ∞ `result_logs.roll_no`

ER description (text you can give to ER tool):

```
STUDENTS(id PK, roll_no UNIQUE, dob, name, gender, category (FK -> CATEGORIES.code), phone, email, percentile, total_marks, air, category_rank, qualified, created_at, updated_at)
CATEGORIES(code PK, name, cutoff_percentile)
RESULT_LOGS(id PK, roll_no FK->STUDENTS.roll_no, viewed_at TIMESTAMP, ip_address)
ADMINS(id PK, username UNIQUE, password_hash, name, email)
```

Normalization: tables are 3NF — data separated by function (students vs categories vs logs).

**Create DB**

**Create tables**

```sql
-- categories (optional)
CREATE TABLE IF NOT EXISTS categories (
  code VARCHAR(10) PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  cutoff_percentile FLOAT DEFAULT NULL
);

-- students
CREATE TABLE IF NOT EXISTS students (
  id INT AUTO_INCREMENT PRIMARY KEY,
  roll_no VARCHAR(30) NOT NULL UNIQUE,
  dob DATE NOT NULL,
  name VARCHAR(100),
  gender ENUM('M','F','O') DEFAULT 'M',
  category VARCHAR(10) DEFAULT NULL,
  phone VARCHAR(20) DEFAULT NULL,
  email VARCHAR(100) DEFAULT NULL,
  percentile FLOAT DEFAULT NULL,
  total_marks FLOAT DEFAULT NULL,
  air INT DEFAULT NULL,
  category_rank INT DEFAULT NULL,
  qualified ENUM('Pass','Fail','Awaiting') DEFAULT 'Awaiting',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (category) REFERENCES categories(code) ON DELETE SET NULL
);

-- result view logs (optional)
CREATE TABLE IF NOT EXISTS result_logs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  roll_no VARCHAR(30) NOT NULL,
  viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  ip_address VARCHAR(45),
  FOREIGN KEY (roll_no) REFERENCES students(roll_no) ON DELETE CASCADE
);

-- admins (optional)
CREATE TABLE IF NOT EXISTS admins (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(100),
  email VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

# 8 — Security & good practises (student level)

* When uploading CSV, validate file type, limit size, and sanitize inputs.
